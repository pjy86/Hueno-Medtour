## 数据库环境策略

### 统一方案：使用 Render PostgreSQL

```
开发阶段（本地调试）：
┌─────────────────┐     ┌─────────────────────────────┐
│  本地 Next.js   │ ──> │  Render PostgreSQL           │
│  (.env.local)   │     │  (开发数据库)                │
└─────────────────┘     └─────────────────────────────┘

正式部署（Render）：
┌─────────────────┐     ┌─────────────────────────────┐
│  Render Web     │ ──> │  Render PostgreSQL           │
│                 │     │  (同一数据库，同一连接)       │
└─────────────────┘     └─────────────────────────────┘
```

### 环境变量配置

**开发环境 `.env.local`：**

```env
DATABASE_URL="postgresql://user:pass@xxx.onrender.com/hntravel_dev?sslmode=require"
NEXT_PUBLIC_SITE_URL="http://localhost:3000"
```

**Render 生产环境（自动注入）：**

```env
DATABASE_URL="postgresql://user:pass@xxx.onrender.com/hntravel"
NEXT_PUBLIC_SITE_URL="https://your-app.onrender.com"
```

### 工作流程

1. **开发时**：本地连接 Render PostgreSQL，使用 `prisma studio` 查看数据
2. **部署时**：Render 自动使用相同的 `DATABASE_URL`，无需更改代码
3. **数据迁移**：`prisma db push` 同步表结构到数据库

### 可选：双数据库环境

如果需要区分开发和生产数据，可以在 Render 创建两个 PostgreSQL 实例：

- `hntravel_dev` - 开发用
- `hntravel_prod` - 生产用

本地开发时 `.env.local` 指向 `hntravel_dev`，Render 部署时使用 `hntravel_prod`。

## 数据库设计

```sql
-- 文案配置表
CREATE TABLE contents (
  id SERIAL PRIMARY KEY,
  key VARCHAR(100) UNIQUE NOT NULL,
  en TEXT,
  zh TEXT,
  id_text TEXT,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 图片配置表
CREATE TABLE images (
  id SERIAL PRIMARY KEY,
  key VARCHAR(100) UNIQUE NOT NULL,
  url TEXT,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 联系方式表
CREATE TABLE contacts (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255),
  email VARCHAR(255),
  phone VARCHAR(100),
  country VARCHAR(100),
  message TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 性能优化

### API 缓存

`/api/cms` 接口配置了缓存响应头：

- `Cache-Control: public, s-maxage=60, stale-while-revalidate=300`
- 60秒缓存，5分钟内可接受 stale 数据

- 自动格式转换（AVIF/WebP）
- 响应式图片加载（`sizes` 属性）
- 懒加载

- `images.unsplash.com`

- `**.render.com`
